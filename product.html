--- inclusion: always --- # Product Overview This project is a browser-based interactive game inspired by the classic **Whack-a-Mole** concept. Players interact with different engineering-themed events that appear randomly on the screen. The objective is to click on **negative engineering events** (bugs, anti-patterns) while avoiding **positive engineering practices**. The game runs for a fixed duration and tracks the player’s score based on their interactions. The application is lightweight and runs entirely in the browser without requiring a backend service. The focus of the product is to provide a simple, responsive, and engaging browser game with clear rules and immediate feedback to the player. ## Key Features - 30-second timed gameplay - Random engineering-themed events appear during gameplay - Players gain points by clicking **negative engineering events** - Players lose points if they click **positive engineering practices** - Score tracking system - Countdown timer visible to the player - Game states: - Idle - Running - Ended - Start and restart functionality - Disable interactions once the game ends - Simple browser-based UI with no build step required ## Target Users - Developers and students interested in interactive programming games - Beginners learning browser-based game mechanics - Users looking for a lightweight web-based mini game ## Project Goals - Build a fully playable browser-based game - Provide clear and responsive gameplay mechanics - Maintain a simple architecture that runs directly in the browser - Ensure the game logic is modular and easy to understand - Support smooth gameplay with minimal UI complexity
