## 🐛 Troubleshooting

### Error: "mvn: command not found"
**Problem:** Maven not installed or not in PATH

**Solution:**
1. Install Maven: https://maven.apache.org/download.cgi
2. Or use package manager:
   - Windows: `choco install maven`
   - Mac: `brew install maven`
   - Linux: `sudo apt install maven`

### Error: "The goal you specified requires a project to execute but there is no POM in this directory"
**Problem:** Running Maven from wrong directory

**Solution:**
```bash
# Make sure you're in the backend directory
cd backend
mvn spring-boot:run
```

### Error: "Failed to execute goal... Could not resolve dependencies"
**Problem:** Maven can't download dependencies

**Solution:**
```bash
# Clear Maven cache and retry
mvn clean install -U

# Or delete .m2 repository and rebuild
rm -rf ~/.m2/repository
mvn clean install
```

### Error: "Source option 5 is no longer supported. Use 7 or later."
**Problem:** Wrong Java version

**Solution:**
```bash
# Check Java version
java -version

# Should be Java 17 or higher
# If not, install Java 17+
```

## 📦 Maven Lifecycle Phases

When you run `mvn install`, Maven executes these phases in order:

1. **validate** - Validate project structure
2. **compile** - Compile source code
3. **test** - Run unit tests
4. **package** - Create JAR file
5. **verify** - Run integration tests
6. **install** - Install to local repository

## 🔧 Customizing pom.xml (Optional)

If you need to modify your `pom.xml`, here are common changes:

### Change Java Version
```xml
<properties>
    <java.version>21</java.version>  <!-- Change from 17 to 21 -->
</properties>
```

### Change Spring Boot Version
```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.3.0</version>  <!-- Update version -->
    <relativePath/>
</parent>
```

### Add New Dependency
```xml
<dependencies>
    <!-- Existing dependencies... -->
    
    <!-- Add new dependency here -->
    <dependency>
        <groupId>group-id</groupId>
        <artifactId>artifact-id</artifactId>
        <version>version</version>
    </dependency>
</dependencies>
```

